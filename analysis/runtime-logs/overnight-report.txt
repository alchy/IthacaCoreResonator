═══════════════════════════════════════════════════════════════════════
 ANALÝZA NaN EXPLOZE (epoch 120 prvního finetune)
═══════════════════════════════════════════════════════════════════════

Příčina: B_net → velké log_B → B = exp(velké) → f_hzs = ∞ pro vysoké k
         valid maska = 0 (nad Nyquistem), ale NaN × 0 = NaN v IEEE 754
         cos(∞) = NaN → otrávení celého audio tensoru → loss = NaN

Projev (dle poslechu uživatele):
  epoch 040: basové driftování (helicopter), pěkné středy a výšky
  epoch 060: pěkné středy/výšky, stále basové driftování
  epoch 080: basové driftování zmizelo, ale bas plochý/nevýrazný
  epoch 100: bas se částečně vrací, zlepšení výšek
  epoch 120: "sumový výstřel – ticho – výstřel" → 100% NaN sampley
  epoch 130+: loss=NaN permanentně

Oprava (commit a2e3626):
  - B = exp(forward_B(mf).clamp(max=0.0)) → B ≤ 1.0 (fyzicky grand piano B ≤ 0.01)
  - valid maska: přidán isfinite() guard
